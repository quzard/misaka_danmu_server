# 更新日志

## v2.7.5

### 新功能

- **浏览器缓存清理入口** — 新增清理浏览器缓存工具，参考 MoviePilot 清理 Cache Storage、注销 Service Worker 并带时间戳刷新；登录页和用户下拉菜单均提供入口。
- **健康检查端点** — 新增 `/api/health` 轻量健康检查接口，供 Docker HEALTHCHECK 和群晖 Container Manager 判断容器状态。

### 安全增强

- **MFA 临时数据容量限制** — 为 `_challenge_store` 与 `_mfa_pending_store` 增加最大容量限制，降低内存膨胀风险。
- **白名单会话缓存容量限制** — 为 IP 白名单免登录会话缓存增加最大容量限制，避免长期运行时缓存无限增长。
- **用户登录令牌保护** — 用户表中的登录 token 改为 SHA256 哈希摘要存储，避免数据库泄露时暴露明文令牌。
- **错误信息收敛** — 登录与 PassKey 相关前端提示不再直接展示后端异常详情，减少敏感信息暴露。
- **TOTP / PassKey 依赖关系修复** — 关闭 TOTP 时自动删除该用户所有 PassKey，并在关闭弹窗中明确提示，避免出现 TOTP 关闭但 PassKey 仍生效的孤立状态。

### 修复

- **HTTP 登录失败问题** — 修复生产环境 CORS 中间件使用空 `allow_origins` 导致浏览器无法读取登录成功响应的问题；生产环境不再启用 CORS，仅开发模式启用本地跨域。
- **群晖容器自动重启优化** — 增加 Docker HEALTHCHECK 与 `/api/health`，减少群晖 Container Manager 误判容器异常导致自动重启的概率。
- **容器内建更新网络保留** — 修复 docker run 模式下重建容器未完整保留网络配置的问题，支持自定义网络与多网络连接。
- **登录页暗色模式背景** — 修复登录页面暗色模式下外层背景色未跟随主题的问题。
- **暗色主题颜色不协调** — 补齐 dark theme 的 fill token 与多个 Ant Design 组件暗色配置，修复输入框、浮层、提示类组件颜色不一致问题。

### 优化

- **MFA 安全流程重构** — 登录流程改为密码验证通过后生成临时 `mfaToken`，再由 MFA 验证接口签发正式 JWT，流程更清晰安全。
- **PassKey 管理体验** — 支持 PassKey 列表展示、最后使用时间、设备重命名和删除操作。
- **登录失败提示优化** — 登录被锁定时前端可展示 429 锁定提示，而不是统一提示密码错误。
- **清理浏览器缓存按钮位置优化** — 登录页清理缓存按钮调整到右上角，避免占用登录卡片内部空间。
- **缓存管理体验优化** — 缓存管理默认显示全部区域，增加键值预览，字段名称中文化。
- **分集过滤日志格式优化** — 分集过滤日志格式改为“黑名单正则匹配：XXX”，更便于定位被过滤原因。

### 维护

- **限流组件更新** — 多次同步更新 rate limit 二进制文件及其签名。
- **应用版本更新** — 更新应用版本至 2.7.6，并同步速率限制模块签名。
